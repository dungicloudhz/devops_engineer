// ── HTTP Function — REST API serverless ──
import com.google.cloud.functions.HttpFunction;
import com.google.cloud.functions.HttpRequest;
import com.google.cloud.functions.HttpResponse;

public class GetOrder implements HttpFunction {
  @Override
  public void service(HttpRequest request, HttpResponse response) throws Exception {
    String orderId = request.getQueryParameters()
        .getOrDefault("orderId", List.of("unknown")).get(0);
    response.setContentType("application/json");
    response.getWriter().write("{\"orderId\":\"" + orderId + "\"}");
  }
}

// ── Pub/Sub Function ──
public class ProcessOrder implements RawBackgroundFunction {
  @Override
  public void accept(String json, Context context) {
    byte[] decoded = Base64.getDecoder().decode(
        JsonParser.parseString(json).getAsJsonObject().get("data").getAsString());
    String message = new String(decoded, StandardCharsets.UTF_8);
    System.out.println("Processing: " + message);
    orderService.process(message);
  }
}

# ── Deploy Cloud Function ──

⚡
Cloud Run vs GKE — chọn gì? Cloud Run: zero ops, scale to 0, tốt cho HTTP services. GKE: phức tạp hơn nhưng full Kubernetes ecosystem, tốt khi cần customization hoặc non-HTTP workloads. Rule of thumb: dùng Cloud Run trước, migrate sang GKE nếu cần.

# ── Cloud Spanner ──
# Relational database + unlimited scale + 5 nines SLA
# Tốt cho: financial systems, gaming, global inventory
$ gcloud spanner instances create spanner-order-prod \
    --config      regional-asia-southeast1 \
    --description "Order Service Spanner" \
    --nodes       3                          # 3 nodes = 6000 QPS write

$ gcloud spanner databases create shopdb \
    --instance spanner-order-prod \
    --ddl "CREATE TABLE orders (
        order_id STRING(36) NOT NULL,
        user_id  STRING(36) NOT NULL,
        status   STRING(20),
        amount   FLOAT64,
        created_at TIMESTAMP OPTIONS (allow_commit_timestamp=true)
    ) PRIMARY KEY (order_id)"

// Java SDK — Spanner
SpannerOptions options = SpannerOptions.newBuilder().build();
Spanner spanner = options.getService();
DatabaseClient client = spanner.getDatabaseClient(
    DatabaseId.of("order-service-prod-12345", "spanner-order-prod", "shopdb"));

// Read-Write Transaction
client.readWriteTransaction().run(txn -> {
    String sql = "INSERT INTO orders (order_id, user_id, status, amount, created_at) " +
                 "VALUES (@orderId, @userId, 'PENDING', @amount, PENDING_COMMIT_TIMESTAMP())";
    Statement stmt = Statement.newBuilder(sql)
        .bind("orderId").to(UUID.randomUUID().toString())
        .bind("userId").to(userId)
        .bind("amount").to(amount).build();
    txn.executeUpdate(stmt);

public boolean isRateLimited(String userId) {

Lộ trình học GCP — từ zero đến production
1

Tạo tài khoản & Google Cloud Console

Free tier: $300 credits 90 ngày. Tạo Project đầu tiên. Cài gcloud CLI, chạy gcloud init. Hiểu Hierarchy: Organization → Folder → Project → Resources.

2

IAM, Service Accounts & gcloud CLI

Tạo Service Account. Assign roles. Bật Application Default Credentials cho local dev. Hiểu Workload Identity để pods không cần key files. Thực hành với gcloud commands.

3

Cloud Run — deploy container đơn giản nhất

Build Docker image, push lên Artifact Registry. Deploy lên Cloud Run. Tích hợp Secret Manager cho DB password. Connect Cloud SQL qua Auth Proxy. Thử traffic splitting.

4

VPC, Cloud SQL & Memorystore

Tạo Custom VPC với private subnets. Cloud SQL PostgreSQL với Private IP. Memorystore Redis. Hiểu Private Google Access — không cần public IP để access Google APIs.

5

GKE Autopilot + Workload Identity

Tạo GKE Autopilot cluster. Setup Workload Identity cho pods. Deploy Spring Boot với Helm. Tích hợp Secret Manager. Cấu hình Cloud SQL Auth Proxy sidecar trong pods.

6

BigQuery, Pub/Sub & Cloud Build CI/CD

Pub/Sub cho event-driven architecture. BigQuery cho analytics. Cloud Build pipeline tự động. GitHub Actions với Workload Identity (passwordless). Cloud Monitoring alerts gửi Slack.
