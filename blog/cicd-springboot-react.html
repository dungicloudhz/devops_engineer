CI/CD cho
Spring Boot & React

Hướng dẫn toàn diện xây dựng pipeline CI/CD production-ready cho dự án full-stack: backend Java Spring Boot và frontend ReactJS — từ cấu trúc project, Dockerfile, testing cho đến deploy tự động.

☕ Java 17 + Spring Boot 3
⚛️ React 18 + Vite
🐳 Docker + Compose
🦊 GitLab CI / 🐙 GitHub Actions
Tại sao cần CI/CD?

Không có CI/CD, mỗi lần deploy là một chuỗi thao tác thủ công dễ sai: build tay, copy file, restart server... CI/CD tự động hóa toàn bộ, giảm rủi ro và giúp team deploy nhiều lần một ngày một cách tự tin.

⚡
Detect lỗi sớm
Auto test sau mỗi commit. Lỗi bị phát hiện ngay, không phải lúc demo với khách.
🔁
Deploy nhất quán
Cùng một process mỗi lần. Loại bỏ "nó chạy trên máy tôi" vĩnh viễn.
📦
Artifact versioned
Mỗi build ra Docker image có tag riêng. Rollback trong 30 giây nếu cần.
✅ Kết quả sau khi hoàn thành guide này
Bạn có một pipeline tự động: khi push code lên main → test chạy → Docker image được build → deploy lên staging → smoke test → notify kết quả. Không cần làm gì thủ công.

Backend · Docker

🐳 Dockerfile Spring Boot — Tối ưu cho CI/CD

Sử dụng multi-stage build và layered JAR để giảm image size và tận dụng Docker layer cache.

dockerfile · backend/Dockerfile
# ══════════════════════════════════════════
# Stage 1: BUILD
# ══════════════════════════════════════════
FROM maven:3.9-eclipse-temurin-17-alpine AS builder

WORKDIR /build

# Bước 1: Copy pom.xml và download dependencies
# Layer này được cache lại nếu pom.xml không thay đổi
# → Build lần sau nhanh hơn rất nhiều
COPY pom.xml .
RUN mvn dependency:go-offline -q

# Bước 2: Copy source và build
COPY src ./src
RUN mvn clean package -DskipTests -q

# Bước 3: Extract layered JAR (Spring Boot 2.3+)
# Dependencies / Spring libs / App code ở layer riêng
RUN java -Djarmode=layertools -jar target/*.jar extract --destination /build/extracted

# ══════════════════════════════════════════
# Stage 2: RUNTIME — Image cuối cùng nhỏ gọn
# ══════════════════════════════════════════
FROM eclipse-temurin:17-jre-alpine

# Security: chạy với non-root user
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
WORKDIR /app

# Copy từng layer riêng — Docker cache hiệu quả nhất
# Layer ít thay đổi nhất → copy trước
COPY --from=builder --chown=appuser:appgroup /build/extracted/dependencies/ ./
COPY --from=builder --chown=appuser:appgroup /build/extracted/spring-boot-loader/ ./
COPY --from=builder --chown=appuser:appgroup /build/extracted/snapshot-dependencies/ ./
COPY --from=builder --chown=appuser:appgroup /build/extracted/application/ ./

USER appuser
EXPOSE 8080

# Health check built-in
HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
    CMD wget -qO- http://localhost:8080/actuator/health || exit 1

ENTRYPOINT ["java", \
    "-XX:+UseContainerSupport", \
    "-XX:MaxRAMPercentage=75.0", \
    "-Djava.security.egd=file:/dev/./urandom", \
    "org.springframework.boot.loader.launch.JarLauncher"]

✅ Lợi ích của Layered JAR
Khi chỉ thay đổi code ứng dụng (không đổi dependencies), Docker chỉ cần rebuild layer cuối cùng (~vài KB). Các layer Spring libs và dependencies (~150MB) được cache → push lên registry chỉ mất vài giây.

🔗

Unit Test mẫu — Service layer

java · UserServiceTest.java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @Mock
    private EmailService emailService;

    @InjectMocks
    private UserService userService;

    @Test
    @DisplayName("createUser: should save user and send welcome email")
    void createUser_success() {
        // Given
        var request = new CreateUserRequest("john@example.com", "John");
        var savedUser = User.builder()
            .id(1L).email("john@example.com").name("John")
            .build();

        Mockito.when(userRepository.save(Mockito.any()))
            .thenReturn(savedUser);

        // When
        var result = userService.createUser(request);

        // Then
        assertThat(result.getId()).isEqualTo(1L);
        assertThat(result.getEmail()).isEqualTo("john@example.com");
        Mockito.verify(emailService).sendWelcome(savedUser);
    }

    @Test
    @DisplayName("createUser: should throw when email already exists")
    void createUser_emailDuplicate_throws() {
        Mockito.when(userRepository.existsByEmail(any())).thenReturn(true);

        assertThatThrownBy(() ->
            userService.createUser(new CreateUserRequest("dup@x.com", "Dup"))
        ).isInstanceOf(DuplicateEmailException.class);
    }
}

Integration Test với Testcontainers

java · UserControllerIT.java (Integration Test)
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@Testcontainers
@ActiveProfiles("test")
class UserControllerIT {

    // Testcontainers tự start/stop PostgreSQL real instance
    @Container
    static PostgreSQLContainer<?> postgres =
        new PostgreSQLContainer<>("postgres:16-alpine")
            .withDatabaseName("testdb")
            .withUsername("test")
            .withPassword("test");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Autowired
    private TestRestTemplate restTemplate;

    @Test
    void createUser_returnsCreated() {
        var request = Map.of("email", "john@test.com", "name", "John");

        var response = restTemplate.postForEntity(
            "/api/v1/users", request, UserResponse.class);

        assertThat(response.getStatusCode()).isEqualTo(HttpStatus.CREATED);
        assertThat(response.getBody().getEmail()).isEqualTo("john@test.com");
    }
}

# ─────────────────────────────────────────────
# Backend CI Pipeline · Spring Boot
# Được include bởi .gitlab-ci.yml chính
# ─────────────────────────────────────────────

be:build:
  stage: build
  image: maven:3.9-eclipse-temurin-17-alpine
  cache:
    key:
      files: [backend/pom.xml]
    paths: [backend/.m2/repository]
    policy: pull-push
  variables:
    MAVEN_OPTS: "-Dmaven.repo.local=$CI_PROJECT_DIR/backend/.m2/repository"
  script:
    - cd backend
    - mvn clean package -DskipTests -q
    - echo "Build size: $(du -sh target/*.jar)"
  artifacts:
    paths: [backend/target/*.jar]
    expire_in: 2 hours
    name: "backend-$CI_COMMIT_SHORT_SHA"
  rules:
    - changes: [backend/**/*]

be:unit-test:
  stage: test
  image: maven:3.9-eclipse-temurin-17-alpine
  cache:
    key:
      files: [backend/pom.xml]
    paths: [backend/.m2/repository]
    policy: pull
  variables:
    MAVEN_OPTS: "-Dmaven.repo.local=$CI_PROJECT_DIR/backend/.m2/repository"
  script:
    - cd backend
    - mvn test -Dgroups="unit" -q
  coverage: '/Total.*?([0-9]{1,3})%/'
  artifacts:
    when: always
    reports:
      junit: backend/target/surefire-reports/**/*.xml
      coverage_report:
        coverage_format: jacoco
        path: backend/target/site/jacoco/jacoco.xml
  needs: [be:build]

be:integration-test:
  stage: test
  image: maven:3.9-eclipse-temurin-17-alpine
  services:
    - name: docker:dind
      alias: docker
  variables:
    DOCKER_HOST: tcp://docker:2376
    DOCKER_TLS_CERTDIR: "/certs"
    MAVEN_OPTS: "-Dmaven.repo.local=$CI_PROJECT_DIR/backend/.m2/repository"
  script:
    - cd backend
    - mvn verify -Dgroups="integration" -q
  artifacts:
    when: always
    reports:
      junit: backend/target/failsafe-reports/**/*.xml
  needs: [be:build]

be:code-quality:
  stage: test
  image: maven:3.9-eclipse-temurin-17-alpine
  variables:
    SONAR_HOST_URL: "$SONAR_URL"
    SONAR_TOKEN: "$SONAR_AUTH_TOKEN"
    MAVEN_OPTS: "-Dmaven.repo.local=$CI_PROJECT_DIR/backend/.m2/repository"
  script:
    - cd backend
    - mvn sonar:sonar -Dsonar.qualitygate.wait=true
  allow_failure: true  # Không block deploy nếu Sonar chưa setup
  needs: [be:unit-test]

be:docker-build:
  stage: package
  image: docker:24-dind
  services:
    - docker:dind
  variables:
    IMAGE: "$CI_REGISTRY_IMAGE/backend:$CI_COMMIT_SHORT_SHA"
    IMAGE_LATEST: "$CI_REGISTRY_IMAGE/backend:latest"
  before_script:
    - docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY
  script:
    - # Build sử dụng cache từ latest image
    - docker pull $IMAGE_LATEST || true
    - docker build
        --cache-from $IMAGE_LATEST
        --build-arg BUILDKIT_INLINE_CACHE=1
        -t $IMAGE -t $IMAGE_LATEST
        backend/
    - docker push $IMAGE
    - docker push $IMAGE_LATEST
    - echo "IMAGE=$IMAGE" >> build.env
  artifacts:
    reports:
      dotenv: build.env
  needs: [be:unit-test, be:integration-test]
  rules:
    - if: '$CI_COMMIT_BRANCH == "main" || $CI_COMMIT_BRANCH == "develop"'
yaml · .github/workflows/backend.yml
name: Backend CI · Spring Boot

on:
  push:
    branches: [main, develop]
    paths: ['backend/**']
  pull_request:
    branches: [main]
    paths: ['backend/**']

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}/backend

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: backend

    steps:
      - uses: actions/checkout@v4

      - name: Setup Java 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: temurin
          cache: maven

      - name: Build (skip tests)
        run: mvn clean package -DskipTests -q

      - name: Run unit tests
        run: mvn test -Dgroups=unit

      - name: Run integration tests
        run: mvn verify -Dgroups=integration

      - name: Upload test reports
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: test-reports
          path: backend/target/surefire-reports/

      - name: Code Coverage Report
        uses: codecov/codecov-action@v4
        with:
          files: backend/target/site/jacoco/jacoco.xml
          flags: backend

  docker-build-push:
    needs: build-and-test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4

      - name: Setup Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push Docker image
        uses: docker/build-push-action@v5
        with:
          context: backend
          push: true
          cache-from: type=registry,ref=${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:buildcache
          cache-to: type=registry,ref=${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:buildcache,mode=max
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest

⚙ vite.config.ts — Cấu hình build tối ưu

import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '')

  return {
    plugins: [react()],

    build: {
      outDir: 'dist',
      sourcemap: mode !== 'production',
      rollupOptions: {
        output: {
          // Code splitting cho vendor libs
          manualChunks: {
            vendor: ['react', 'react-dom'],
            router: ['react-router-dom'],
            query: ['@tanstack/react-query'],
          }
        }
      }
    },

    server: {
      port: 3000,
      // Proxy API calls đến backend trong dev
      proxy: {
        '/api': {
          target: env.VITE_API_URL || 'http://localhost:8080',
          changeOrigin: true,
        }
      }
    },

    test: {
      globals: true,
      environment: 'jsdom',
      setupFiles: ['./src/test-setup.ts'],
      coverage: {
        provider: 'v8',
        reporter: ['text', 'lcov', 'html'],
        thresholds: {
          lines: 70,
          functions: 70,
        }
      }
    }
  }
})

    # Health check endpoint cho CI/CD
    location /health {
        access_log off;
        return 200 '{"status":"ok"}';
        add_header Content-Type application/json;
    }

    # Static assets: cache dài (có content hash trong tên file)
    location ~* \.(js|css|png|jpg|gif|ico|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API proxy đến backend (optional - dùng khi BE và FE cùng domain)

        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

🔵🟢 Blue/Green Deploy script cho Docker Compose

bash · scripts/deploy-bluegreen.sh
#!/bin/bash
# Blue/Green deploy cho full-stack app
# Usage: ./deploy-bluegreen.sh BE_IMAGE FE_IMAGE
set -euo pipefail

BE_IMAGE="$1"
FE_IMAGE="$2"
COMPOSE_FILE="docker-compose.prod.yml"

log() { echo "[$(date +'%H:%M:%S')] $*"; }

# Xác định slot hiện tại (blue hoặc green)
CURRENT_SLOT=$(cat /tmp/current-slot 2>/dev/null || echo "blue")
if [ "$CURRENT_SLOT" = "blue" ]; then
  NEXT_SLOT="green"; NEXT_BE_PORT=8081; NEXT_FE_PORT=3001
else
  NEXT_SLOT="blue";  NEXT_BE_PORT=8080; NEXT_FE_PORT=3000
fi

log "🚀 Deploying to slot: $NEXT_SLOT"
log "   Backend  : $BE_IMAGE"
log "   Frontend : $FE_IMAGE"

# Start new slot containers
IMAGE=$BE_IMAGE FE_IMAGE=$FE_IMAGE \
  docker compose -f $COMPOSE_FILE \
  -p app-$NEXT_SLOT up -d

# Health check với retry
log "⏳ Health checking $NEXT_SLOT..."
for i in $(seq 1 12); do
  BE_H=$(curl -sf http://localhost:$NEXT_BE_PORT/actuator/health | grep -c "UP" || echo 0)
  FE_H=$(curl -so /dev/null -w "%{http_code}" http://localhost:$NEXT_FE_PORT/health)
  if [ "$BE_H" -gt 0 ] && [ "$FE_H" = "200" ]; then
    log "✅ Health check passed on attempt $i"
    break
  fi
  if [ "$i" = "12" ]; then log "❌ Health check failed"; exit 1; fi
  sleep 10
done

# Switch Nginx upstream đến slot mới
log "🔄 Switching traffic to $NEXT_SLOT..."
echo "upstream backend { server localhost:${NEXT_BE_PORT}; }" > /etc/nginx/conf.d/upstream.conf
echo "upstream frontend { server localhost:${NEXT_FE_PORT}; }" >> /etc/nginx/conf.d/upstream.conf
nginx -s reload

# Stop slot cũ
log "🛑 Stopping old slot: $CURRENT_SLOT"
docker compose -f $COMPOSE_FILE -p app-$CURRENT_SLOT down || true

echo "$NEXT_SLOT" > /tmp/current-slot
log "🎉 Deploy completed! Active slot: $NEXT_SLOT"

🐛 Troubleshooting thường gặp

❌ Maven build thất bại trong CI nhưng chạy OK local

⚠ Nguyên nhân phổ biến
CI runner dùng JDK version khác. CI không có ~/.m2/settings.xml. Test cần database nhưng không có service.
bash · Debug trong CI
# Thêm vào pipeline script để debug
java -version
mvn -version
echo "JAVA_HOME=$JAVA_HOME"
echo "PATH=$PATH"

# Chạy với verbose output
mvn test -X 2>&1 | tail -100

❌ React build thành công nhưng app blank trên browser

  • Base path sai: Nếu deploy vào subdirectory, cần set base trong vite.config.ts và homepage trong package.json.
  • API URL hardcode: Kiểm tra VITE_API_URL có được inject đúng lúc build không. Các biến VITE_* được bake vào bundle lúc build.
  • Nginx không có SPA fallback: Đảm bảo try_files $uri $uri/ /index.html có trong nginx.conf.
  • CORS: Kiểm tra Spring Boot đã config @CrossOrigin hoặc CorsConfiguration cho đúng origin của frontend.

❌ Docker image quá lớn

bash · Phân tích layer image
# Xem kích thước từng layer
docker history my-app:latest --format "table {{.CreatedBy}}\t{{.Size}}"

# Dùng dive để phân tích chi tiết
docker run --rm -it \
  -v /var/run/docker.sock:/var/run/docker.sock \
  wagoodman/dive my-app:latest

# Kiểm tra nếu quên .dockerignore
docker build --no-cache -t test . 2>&1 | grep "Sending build context"

❌ Container backend không kết nối được database

💡 Networking trong Docker Compose
Containers trong cùng Compose network gọi nhau qua service name, không phải localhost. Backend connect đến Postgres bằng URL jdbc:postgresql://postgres:5432/mydb — trong đó postgres là tên service trong compose.

❌ Pipeline mất quá nhiều thời gian

  • Cache miss thường xuyên: Kiểm tra cache key — nên dựa trên pom.xml hoặc package-lock.json, không phải branch.
  • Integration test chậm: Testcontainers startup lâu. Dùng @DirtiesContext(classMode = NEVER) để reuse Spring context.
  • Docker build không dùng cache: Đảm bảo --cache-from pull được image trước khi build. Dùng BuildKit: DOCKER_BUILDKIT=1.
  • npm install chậm: Dùng npm ci thay npm install. Cache node_modules bằng CI cache keyed theo package-lock.json.
