Jenkins
từ A đến Z

Hướng dẫn toàn diện về Jenkins — automation server phổ biến nhất thế giới. Từ cài đặt, cấu hình, viết Jenkinsfile cho đến tối ưu pipeline production.

Tổng quan

Jenkins là open-source automation server viết bằng Java, ra đời từ 2004. Với hơn 1800+ plugins, Jenkins có thể tích hợp với hầu hết mọi tool trong hệ sinh thái phát triển phần mềm.

🌍
Phổ biến nhất
Được dùng tại hàng triệu tổ chức toàn cầu. Community lớn, tài liệu phong phú.
🔌
1800+ Plugins
Tích hợp với mọi tool: Git, Docker, Kubernetes, Slack, AWS, Azure, GCP...
⚙
Cực kỳ linh hoạt
Pipeline as Code với Groovy. Chạy bất kỳ đâu: bare metal, VM, Docker, K8s.
🔓
100% Open Source
Miễn phí hoàn toàn. Self-hosted, toàn quyền kiểm soát data và pipeline.
🌿
Multibranch
Tự động phát hiện và tạo pipeline cho mọi branch và Pull Request.
📚
Shared Libraries
Tái sử dụng Groovy code giữa nhiều pipeline. DRY principle cho CI/CD.
ℹ Jenkins vs Alternatives
Jenkins mạnh nhất về tính linh hoạt và plugin ecosystem. Trade-off là cần cấu hình nhiều hơn so với GitLab CI hay GitHub Actions. Phù hợp nhất với các tổ chức lớn cần customize sâu và self-host hoàn toàn.

Cài đặt

⚙ Cài đặt Jenkins với Docker

Cách khuyến nghị nhất: chạy Jenkins trong Docker để dễ quản lý, upgrade, và backup.

📄 docker-compose.yml

yaml · docker-compose.yml
version: '3.8'

networks:
  jenkins-net:
    driver: bridge

volumes:
  jenkins-home:
  jenkins-docker-certs:

services:
  # Docker-in-Docker — cho phép Jenkins build Docker images
  docker:
    image: docker:dind
    container_name: jenkins-docker
    privileged: true
    restart: unless-stopped
    networks:
      jenkins-net:
        aliases: [docker]
    environment:
      - DOCKER_TLS_CERTDIR=/certs
    volumes:
      - jenkins-docker-certs:/certs/client
      - jenkins-home:/var/jenkins_home
    ports:
      - "2376:2376"

  jenkins:
    image: jenkins/jenkins:lts-jdk17
    container_name: jenkins
    restart: unless-stopped
    networks: [jenkins-net]
    ports:
      - "8080:8080"    # Web UI
      - "50000:50000"  # Agent JNLP port
    environment:
      - DOCKER_HOST=tcp://docker:2376
      - DOCKER_CERT_PATH=/certs/client
      - DOCKER_TLS_VERIFY=1
      # Skip setup wizard (optional)
      - JAVA_OPTS=-Djenkins.install.runSetupWizard=false
    volumes:
      - jenkins-home:/var/jenkins_home
      - jenkins-docker-certs:/certs/client:ro
    depends_on:
      - docker
bash · Khởi động
# Start Jenkins
docker compose up -d

# Xem logs, chờ "Jenkins is fully up and running"
docker logs -f jenkins

# Lấy initial admin password
docker exec jenkins cat /var/jenkins_home/secrets/initialAdminPassword

# Truy cập UI
# → http://localhost:8080
✅ Tại sao dùng Docker-in-Docker?
DinD cho phép Jenkins build Docker images bên trong container. jenkins-docker là daemon Docker riêng biệt, Jenkins kết nối qua TCP TLS — bảo mật hơn mount /var/run/docker.sock trực tiếp.
bash · Ubuntu / Debian
# 1. Cài Java 17
sudo apt update && sudo apt install -y fontconfig openjdk-17-jdk

# 2. Thêm Jenkins APT repository

Cấu hình

🔧 Setup lần đầu tiên

1
Unlock Jenkins
Vào http://localhost:8080 → nhập initialAdminPassword từ lệnh phía trên.
2
Cài plugins ban đầu
Chọn "Install suggested plugins" — Jenkins tự cài các plugin phổ biến nhất. Mất 2–5 phút tuỳ tốc độ mạng.
3
Tạo Admin Account
Điền đầy đủ: Username, Password, Full name, Email. Đây là tài khoản admin chính, ghi nhớ cẩn thận.
4
Cấu hình Jenkins URL
Mặc định là http://localhost:8080/. Nếu có domain, đổi thành https://jenkins.yourcompany.com/ — quan trọng cho webhook và notifications.
5
Cấu hình System
Manage Jenkins → System
Thiết lập: System Message, Executors (số build parallel), Jenkins URL, Email server.
6
Cấu hình Global Tools
Manage Jenkins → Tools
Cấu hình JDK, Maven, NodeJS, Git, Docker... Có thể để Jenkins tự install hoặc chỉ path có sẵn trên agent.

⚙ Cấu hình số Executor // Manage Jenkins → System

⚠ Quan trọng về Controller Executor
Đặt Number of executors = 0 trên Controller node. Controller chỉ nên điều phối, không chạy build. Để build chạy trên dedicated Agents. Đây là best practice bảo mật và performance.
groovy · Cấu hình qua Script Console (nếu cần)
// Manage Jenkins → Script Console
// Đặt số executor của controller về 0
import jenkins.model.*

def jenkins = Jenkins.instance
jenkins.setNumExecutors(0)
jenkins.save()
println "Done! Executors set to 0"

Thêm Credential qua UI

Manage Jenkins → Credentials → System → Global credentials → Add Credentials

Dùng Credentials trong Jenkinsfile

groovy · Các cách dùng credentials
pipeline {
    agent any

    environment {
        // Cách 1: Bind Secret Text → biến env đơn giản
        SLACK_TOKEN = credentials('slack-token')

        // Cách 2: Username + Password → tạo 2 biến: _USR và _PSW
        DOCKER_CREDS = credentials('docker-registry-creds')
        // → DOCKER_CREDS_USR = username
        // → DOCKER_CREDS_PSW = password
    }

    stages {
        stage('Use Credentials') {
            steps {
                // Cách 3: withCredentials block — scope hẹp hơn, an toàn hơn
                withCredentials([
                    usernamePassword(
                        credentialsId: 'github-credentials',
                        usernameVariable: 'GIT_USER',
                        passwordVariable: 'GIT_TOKEN'
                    )
                ]) {
                    sh '''
                        git config user.email "$GIT_USER"
                        git push https://$GIT_USER:$GIT_TOKEN@github.com/...
                    '''
                }

                // Cách 4: SSH key
                withCredentials([
                    sshUserPrivateKey(
                        credentialsId: 'deploy-ssh-key',
                        keyFileVariable: 'SSH_KEY',
                        usernameVariable: 'SSH_USER'
                    )
                ]) {
                    sh 'ssh -i $SSH_KEY $SSH_USER@server.com "deploy.sh"'
                }

                // Cách 5: Secret file
                withCredentials([
                    file(credentialsId: 'kube-config', variable: 'KUBECONFIG')
                ]) {
                    sh 'kubectl apply -f deployment.yaml'
                }
            }
        }
    }
}
🚨 Không bao giờ làm
Không dùng echo $SECRET — Jenkins sẽ mask trong logs nhưng đây vẫn là bad practice. Không hardcode credential ID vào Jenkinsfile — dùng parameter hoặc environment variable.
Agents

🤖 Agents & Nodes

Agent là nơi thực thi build. Có nhiều loại agent khác nhau cho các use case khác nhau.

🖥
Permanent Agent (SSH)
Server vật lý hoặc VM được cấu hình cố định. Kết nối qua SSH. Phù hợp cho build cần môi trường đặc biệt.
🐳
Docker Agent
Mỗi build chạy trong Docker container sạch. Không cần cài tool trên host. Phổ biến nhất hiện tại.
☸
Kubernetes Agent
Pod được tạo tự động khi có build, xóa sau khi xong. Scale tự động, không tốn tài nguyên idle.
☁
Cloud Agent (EC2/GCP)
Spin up cloud VM khi cần, terminate sau build. Cost-efficient cho workload không đều.

Thêm SSH Agent

1
Chuẩn bị máy Agent
Cài Java (≥11), tạo user jenkins, thêm SSH public key của Jenkins Controller vào ~/.ssh/authorized_keys.
2
Thêm Node trong Jenkins
Manage Jenkins → Nodes → New Node
Điền: Name, Remote root directory (/home/jenkins/agent), Labels, Launch method = SSH.
3
Thêm SSH Credential
Chọn SSH credential đã tạo (private key của Jenkins). Save và Launch Agent để test kết nối.

Docker Agent trong Jenkinsfile

groovy · Docker agent examples
// ── Global Docker agent ──
pipeline {
    agent {
        docker {
            image 'maven:3.9-eclipse-temurin-17-alpine'
            args  '-v $HOME/.m2:/root/.m2'  // mount Maven cache
        }
    }
    stages {
        stage('Build') {
            steps { sh 'mvn clean package' }
        }
    }
}

// ── Per-stage Docker agent (khác nhau theo stage) ──
pipeline {
    agent none  // không global agent

    stages {
        stage('Backend Build') {
            agent { docker { image 'maven:3.9-eclipse-temurin-17-alpine' } }
            steps { sh 'mvn clean package' }
        }
        stage('Frontend Build') {
            agent { docker { image 'node:20-alpine' } }
            steps { sh 'npm ci && npm run build' }
        }
        stage('Docker Build') {
            agent { label 'docker-host' }  // chạy trên node có Docker
            steps { sh 'docker build -t my-app .' }
        }
    }
}

// ── Kubernetes agent ──
pipeline {
    agent {
        kubernetes {
            yaml '''
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: maven
    image: maven:3.9-eclipse-temurin-17
    command: [sleep, infinity]
  - name: docker
    image: docker:dind
    securityContext:
      privileged: true
'''
            defaultContainer 'maven'
        }
    }
    stages {
        stage('Build') {
            steps { sh 'mvn package' }
        }
        stage('Docker') {
            steps {
                container('docker') {
                    sh 'docker build -t app .'
                }
            }
        }
    }
}
Pipeline

🔄 Các loại Pipeline

📝
Declarative Pipeline ⭐
Cú pháp có cấu trúc rõ ràng, dễ đọc, dễ validate. Khuyến nghị cho hầu hết dự án. Linter hỗ trợ tốt.

Scripted Pipeline

Quét repo, tự động tạo job cho mỗi branch có Jenkinsfile. Auto-delete khi branch bị xóa.
🏗
Organization Folder
Quét toàn bộ GitHub Org hoặc GitLab Group. Tự động discover mọi repo có Jenkinsfile.

Core

📝 Jenkinsfile toàn diện từ A–Z

Dưới đây là Jenkinsfile Declarative đầy đủ với mọi tính năng thường dùng được giải thích chi tiết:

groovy · Jenkinsfile — Declarative Pipeline hoàn chỉnh
/* ═══════════════════════════════════════════════
   Jenkinsfile · Declarative Pipeline
   Full-featured example
═══════════════════════════════════════════════ */

pipeline {

    // ── AGENT: Nơi chạy pipeline ──────────────────
    agent {
        docker {
            image 'maven:3.9-eclipse-temurin-17-alpine'
            args  '-v /root/.m2:/root/.m2 --network host'
        }
    }

    // ── OPTIONS: Tùy chọn pipeline ────────────────
    options {
        buildDiscarder(logRotator(
            numToKeepStr: '20',       // giữ 20 build gần nhất
            daysToKeepStr: '30'      // giữ trong 30 ngày
        ))
        timeout(time: 30, unit: 'MINUTES')  // timeout toàn pipeline
        disableConcurrentBuilds()              // không chạy parallel với chính nó
        timestamps()                           // thêm timestamp vào log
        ansiColor('xterm')                    // màu trong log (cần AnsiColor plugin)
        skipDefaultCheckout()                  // tự control checkout
    }

    // ── PARAMETERS: Input từ người dùng ──────────
    parameters {
        booleanParam(
            name: 'SKIP_TESTS',
            defaultValue: false,
            description: 'Skip test stages?'
        )
        choice(
            name: 'DEPLOY_ENV',
            choices: ['staging', 'production'],
            description: 'Target environment'
        )
        string(
            name: 'IMAGE_TAG',
            defaultValue: '',
            description: 'Docker image tag (empty = commit SHA)'
        )
    }

    // ── ENVIRONMENT: Biến môi trường ─────────────
    environment {
        APP_NAME     = 'my-app'
        REGISTRY     = 'registry.mycompany.com'
        IMAGE_TAG    = "${params.IMAGE_TAG ?: env.GIT_COMMIT.take(8)}"
        IMAGE_FULL   = "${REGISTRY}/${APP_NAME}:${IMAGE_TAG}"
        MAVEN_OPTS   = '-Xmx1024m -XX:MaxPermSize=256m'

        // Bind credentials an toàn
        DOCKER_CREDS  = credentials('docker-registry-creds')
        SONAR_TOKEN   = credentials('sonarqube-token')
        SLACK_CHANNEL = '#deployments'
    }

    // ── STAGES ────────────────────────────────────
    stages {

        // 1. CHECKOUT
        stage('Checkout') {
            steps {
                checkout scm
                script {
                    // Lưu thông tin commit
                    env.GIT_AUTHOR   = sh(script: 'git log -1 --format="%an"', returnStdout: true).trim()
                    env.GIT_MESSAGE  = sh(script: 'git log -1 --format="%s"', returnStdout: true).trim()
                    env.GIT_BRANCH_NAME = env.GIT_BRANCH.replaceAll('origin/', '')
                }
                echo "Branch: ${env.GIT_BRANCH_NAME}"
                echo "Commit by: ${env.GIT_AUTHOR} — ${env.GIT_MESSAGE}"
            }
        }

        // 2. BUILD
        stage('Build') {
            steps {
                sh '''
                    echo "📦 Building ${APP_NAME}..."
                    mvn clean package -DskipTests -q
                    echo "✅ Build size: $(du -sh target/*.jar)"
                '''
            }
            post {
                success {
                    archiveArtifacts artifacts: 'target/*.jar', fingerprint: true
                }
            }
        }

        // 3. TEST (skip nếu params.SKIP_TESTS = true)
        stage('Unit Tests') {
            when { expression { !params.SKIP_TESTS } }
            steps {
                sh 'mvn test -Dgroups=unit'
            }
            post {
                always {
                    junit testResults: 'target/surefire-reports/**/*.xml', allowEmptyResults: true
                    jacoco(
                        execPattern: 'target/jacoco.exec',
                        classPattern: 'target/classes',
                        sourcePattern: 'src/main/java',
                        minimumLineCoverage: '70',
                        changeBuildStatus: true
                    )
                }
            }
        }

        // 4. PARALLEL STAGES — Chạy song song
        stage('Quality Gates') {
            when { expression { !params.SKIP_TESTS } }
            parallel {
                stage('Integration Tests') {
                    agent { label 'docker-host' }
                    steps {
                        sh 'mvn verify -Dgroups=integration -q'
                    }
                    post {
                        always {
                            junit 'target/failsafe-reports/**/*.xml'
                        }
                    }
                }
                stage('SonarQube Analysis') {
                    steps {
                        withSonarQubeEnv('SonarQube') {
                            sh 'mvn sonar:sonar'
                        }
                        timeout(time: 5, unit: 'MINUTES') {
                            waitForQualityGate abortPipeline: true
                        }
                    }
                }
                stage('Security Scan') {
                    steps {
                        sh '''
                            docker run --rm \
                              -v /var/run/docker.sock:/var/run/docker.sock \
                              aquasec/trivy fs --exit-code 1 \
                              --severity CRITICAL .
                        '''
                    }
                }
            }
        }

        // 5. DOCKER BUILD & PUSH
        stage('Docker Build & Push') {
            when {
                anyOf {
                    branch 'main'
                    branch 'develop'
                    branch 'release/*'
                }
            }
            agent { label 'docker-host' }
            steps {
                script {
                    docker.withRegistry("https://${REGISTRY}", 'docker-registry-creds') {
                        def img = docker.build(IMAGE_FULL, "--build-arg APP_VERSION=${IMAGE_TAG} .")
                        img.push()
                        img.push('latest')

                        // Tag thêm branch name
                        img.push(env.GIT_BRANCH_NAME.replaceAll('/', '-'))
                    }
                }
                echo "✅ Pushed: ${IMAGE_FULL}"
            }
        }

        // 6. DEPLOY
        stage('Deploy') {
            when { branch 'main' }
            input {  // Yêu cầu approve thủ công
                message "Deploy to ${params.DEPLOY_ENV}?"
                ok      "Yes, Deploy!"
                submitter 'admin,devlead'
            }
            steps {
                withCredentials([
                    sshUserPrivateKey(credentialsId: 'deploy-ssh-key',
                                     keyFileVariable: 'SSH_KEY',
                                     usernameVariable: 'SSH_USER')
                ]) {
                    sh """
                        ssh -i $SSH_KEY -o StrictHostKeyChecking=no \\
                            $SSH_USER@${params.DEPLOY_ENV}.myapp.com \\
                            'cd /app && \\
                             export IMAGE=${IMAGE_FULL} && \\
                             docker compose pull && \\
                             docker compose up -d --remove-orphans'
                    """
                }
            }
        }

        // 7. SMOKE TEST sau deploy
        stage('Smoke Test') {
            when { branch 'main' }
            steps {
                retry(3) {
                    sleep(10)
                    sh """
                        STATUS=\$(curl -s -o /dev/null -w "%{http_code}" \\
                            https://${params.DEPLOY_ENV}.myapp.com/actuator/health)
                        [ "\$STATUS" = "200" ] && echo "✅ App healthy" || exit 1
                    """
                }
            }
        }
    }

    // ── POST: Chạy sau khi pipeline kết thúc ─────
    post {
        always {
            cleanWs()  // Dọn workspace
        }
        success {
            slackSend(
                channel: env.SLACK_CHANNEL,
                color: 'good',
                message: """✅ *BUILD SUCCESS* · ${env.APP_NAME}
Branch: `${env.GIT_BRANCH_NAME}` · Tag: `${env.IMAGE_TAG}`
Author: ${env.GIT_AUTHOR}
<${env.BUILD_URL}|View Build>"""
            )
        }
        failure {
            slackSend(
                channel: env.SLACK_CHANNEL,
                color: 'danger',
                message: """❌ *BUILD FAILED* · ${env.APP_NAME}
Stage: `${env.STAGE_NAME}` · Branch: `${env.GIT_BRANCH_NAME}`
<${env.BUILD_URL}console|View Logs>"""
            )
            emailext(
                subject: "[Jenkins] FAILED: ${env.APP_NAME} #${env.BUILD_NUMBER}",
                body: "Build failed! View: ${env.BUILD_URL}",
                to: 'team@mycompany.com'
            )
        }
        unstable {
            slackSend(channel: env.SLACK_CHANNEL, color: 'warning',
                      message: "⚠ Build UNSTABLE: ${env.APP_NAME} #${env.BUILD_NUMBER}")
        }
        aborted {
            echo 'Pipeline was aborted.'
        }
    }
}

/**
 * Build và push Docker image
 * @param config Map với keys: image, registry, credentialsId, tags
 */
def call(Map config = [:]) {
    def image      = config.get('image')      ?: error('image is required')
    def registry   = config.get('registry')   ?: 'registry.mycompany.com'
    def credId     = config.get('credId')     ?: 'docker-registry-creds'
    def extraTags  = config.get('tags')      ?: []
    def dockerfile = config.get('dockerfile') ?: 'Dockerfile'
    def context    = config.get('context')    ?: '.'

    def shortSha = sh(script: 'git rev-parse --short HEAD', returnStdout: true).trim()
    def fullImage = "${registry}/${image}:${shortSha}"

    docker.withRegistry("https://${registry}", credId) {
        def img = docker.build(fullImage, "-f ${dockerfile} ${context}")
        img.push()
        img.push('latest')

        extraTags.each { tag ->
            img.push(tag)
        }
    }

    echo "✅ Pushed: ${fullImage}"
    return fullImage
}

groovy · Docker build, tag, push
stage('Docker Build & Push') {
    steps {
        script {
            def shortSha = sh(script: 'git rev-parse --short HEAD', returnStdout: true).trim()
            def registry = 'registry.mycompany.com'
            def imageName = "${registry}/my-app"

            // Build với cache từ latest
            sh """
                docker pull ${imageName}:latest || true
                docker build \\
                    --cache-from ${imageName}:latest \\
                    --build-arg BUILDKIT_INLINE_CACHE=1 \\
                    --build-arg APP_VERSION=${shortSha} \\
                    -t ${imageName}:${shortSha} \\
                    -t ${imageName}:latest \\
                    .
            """

            // Push với credentials
            docker.withRegistry("https://${registry}", 'registry-creds') {
                sh "docker push ${imageName}:${shortSha}"
                sh "docker push ${imageName}:latest"
            }

            // Lưu image name để dùng ở stages sau
            env.DOCKER_IMAGE = "${imageName}:${shortSha}"
            echo "📦 Image pushed: ${env.DOCKER_IMAGE}"

            // Cleanup local images
            sh "docker rmi ${imageName}:${shortSha} ${imageName}:latest || true"
        }
    }
}

📋 Configuration as Code (JCasC)

Jenkins Configuration as Code plugin cho phép định nghĩa toàn bộ cấu hình Jenkins dưới dạng YAML file — Infrastructure as Code cho CI server.

✅ Lợi ích của JCasC
Reproducible setup — clone Jenkins server trong 5 phút. Version control cấu hình. Không bao giờ mất config khi reset. Dễ dàng audit thay đổi.
yaml · jenkins.yaml — JCasC config đầy đủ
jenkins:
  systemMessage: "🚀 Jenkins CI/CD · Managed by JCasC"
  numExecutors: 0         # controller không chạy build
  scmCheckoutRetryCount: 3
  mode: EXCLUSIVE

  securityRealm:
    local:
      allowsSignup: false
      users:
        - id: admin
          name: Administrator
          password: "${JENKINS_ADMIN_PASS}"  # từ env var

  authorizationStrategy:
    roleBased:
      roles:
        global:
          - name: admin
            description: Jenkins administrators
            permissions: [Overall/Administer]
            assignments: [admin]
          - name: developer
            description: Developers
            permissions: [Overall/Read, Job/Build, Job/Read, View/Read]
            assignments: []

  nodes:
    - permanent:
        name: docker-agent-01
        labelString: docker-host linux
        numExecutors: 4
        remoteFS: /home/jenkins/agent
        launcher:
          ssh:
            host: agent01.internal
            port: 22
            credentialsId: agent-ssh-key

credentials:
  system:
    domainCredentials:
      - credentials:
          - usernamePassword:
              id: docker-registry-creds
              username: "${REGISTRY_USER}"
              password: "${REGISTRY_PASS}"
              description: Docker Registry Credentials
          - string:
              id: slack-token
              secret: "${SLACK_TOKEN}"
              description: Slack Bot Token

unclassified:
  slackNotifier:
    teamDomain: mycompany
    tokenCredentialId: slack-token
    room: '#deployments'

  sonarGlobalConfiguration:
    buildWrapperEnabled: true
    installations:
      - name: SonarQube
        serverUrl: http://sonarqube:9000
        credentialsId: sonarqube-token

tool:
  jdk:
    installations:
      - name: JDK-17
        home: /usr/local/openjdk-17
  maven:
    installations:
      - name: Maven-3.9
        properties:
          - installSource:
              installers:
                - maven:
                    id: "3.9.5"
  nodejs:
    installations:
      - name: Node-20
        properties:
          - installSource:
              installers:
                - nodeJSInstaller:
                    id: "20.10.0"

🐛 Troubleshooting thường gặp

❌ "java.lang.OutOfMemoryError: Java heap space"

⚠ Jenkins thiếu RAM
Tăng heap size. Xóa old builds. Reduce number of executors. Chuyển builds về agents.
bash · Fix OOM
# Tăng heap trong Docker Compose environment:
JAVA_OPTS=-Xms512m -Xmx4g -XX:+UseG1GC

# Dọn build history qua Script Console:
# Manage Jenkins → Script Console
Jenkins.instance.allItems(Job).each { job ->
    job.builds.each { build ->
        if (build.number < (job.lastBuild.number - 20)) {
            build.delete()
        }
    }
}

# Manage Jenkins → System Information → Memory Usage

❌ Pipeline stuck ở "Waiting for next available executor"

  • Không có agent nào available: Kiểm tra Manage Jenkins → Nodes — agent có Online không? Số executor đã đầy chưa?
  • Label mismatch: Pipeline yêu cầu agent { label 'docker-host' } nhưng không có node nào có label đó.
  • Controller executor = 0: Nếu không có agent nào, đặt controller executor > 0 tạm thời để test.

❌ "Host key verification failed" khi SSH deploy

groovy · Fix SSH host verification
// Thêm -o StrictHostKeyChecking=no (dev/staging only) sh 'ssh -o StrictHostKeyChecking=no -i $SSH_KEY user@host "cmd"' // Hoặc thêm known_hosts an toàn hơn sh ''' mkdir -p ~/.ssh ssh-keyscan -H staging.myapp.com >> ~/.ssh/known_hosts ssh -i $SSH_KEY user@staging.myapp.com "deploy.sh" '''

❌ Groovy script bị từ chối (Script Security)

Manage Jenkins → In-process Script Approval → Approve pending signatures

Hoặc chuyển logic sang Shared Library (được trust nhiều hơn). Hoặc dùng Declarative thay Scripted để tránh sandbox issues.

❌ Build chậm không rõ nguyên nhân

groovy · Debug timing
// Wrap stages với timestamps để xem bước nào chậm timestamps { stage('Debug Stage') { steps { sh ''' date && echo "Start build" mvn clean package date && echo "End build" ''' } } } // Xem Pipeline timing trong Blue Ocean hoặc Stage View // Mỗi stage hiển thị duration riêng → dễ phát hiện bottleneck
