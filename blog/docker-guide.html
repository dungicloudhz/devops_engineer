Images — Kéo, xem và quản lý

docker pull, image ls, rmi, tag, save/load — làm chủ image management

Image commands thực tế
docker image commandsImages
# ── Pull images ──
$ docker pull nginx                        # latest tag
$ docker pull nginx:1.25-alpine           # specific version
$ docker pull eclipse-temurin:21-jdk-jammy # Java 21 JDK
$ docker pull postgres:16-alpine           # PostgreSQL 16

# ── Xem images ──
$ docker images                              # danh sách images
$ docker image ls --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"
$ docker image inspect nginx:latest        # metadata đầy đủ JSON
$ docker history my-app:1.0               # xem từng layer + size

# ── Tag và rename ──
$ docker tag my-app:latest registry.company.com/backend/my-app:v1.2.3
$ docker tag my-app:latest my-app:stable

# ── Xóa images ──
$ docker rmi nginx:1.24                    # xóa image
$ docker image prune                         # xóa dangling images (none)
$ docker image prune -a                      # xóa tất cả không dùng

# ── Save/Load (offline transfer) ──
$ docker save my-app:1.0 -o my-app-1.0.tar  # export sang file
$ docker load -i my-app-1.0.tar              # import từ file
$ docker save my-app | gzip > my-app.tar.gz # nén luôn

# ── Scan vulnerabilities ──
$ docker scout cves my-app:latest           # Docker Scout (built-in)
$ trivy image my-app:latest                 # Trivy scan

# Ví dụ 1: Chạy Nginx web server
$ docker run \
    --name nginx-prod \        # đặt tên container
    -d \                         # detached (chạy nền)
    -p 80:80 \                   # host:container port
    -p 443:443 \
    -v /etc/nginx/nginx.conf:/etc/nginx/nginx.conf:ro \  # mount file config
    --restart unless-stopped \  # tự restart khi crash
    nginx:1.25-alpine

# Ví dụ 2: Chạy PostgreSQL với data volume
$ docker run \
    --name postgres \
    -d \
    -p 5432:5432 \
    -e POSTGRES_DB=shopdb \          # environment variable
    -e POSTGRES_USER=admin \
    -e POSTGRES_PASSWORD=secret \
    -v pgdata:/var/lib/postgresql/data \  # named volume
    --network backend-net \      # join network
    --memory 512m \             # limit RAM
    --cpus 1.0 \                 # limit CPU
    postgres:16-alpine

# Ví dụ 3: Chạy Spring Boot app
$ docker run \
    --name order-service \
    -d \
    -p 8080:8080 \
    -e SPRING_PROFILES_ACTIVE=prod \
    -e DB_URL=jdbc:postgresql://postgres:5432/shopdb \
    --env-file .env \              # load từ file .env
    --network backend-net \
    --restart on-failure:3 \     # restart tối đa 3 lần
    --health-cmd "curl -f http://localhost:8080/actuator/health" \
    --health-interval 30s \
    --health-timeout 10s \
    --health-retries 3 \
    registry.company.com/order-service:v2.1.0

# Ví dụ 4: Interactive shell (debug)
$ docker run -it --rm ubuntu:22.04 bash  # -it=interactive, --rm=auto delete
$ docker run -it --rm eclipse-temurin:21 java -version

$ docker stop order-service                 # SIGTERM → grace period → SIGKILL
$ docker stop -t 30 order-service           # đợi 30s trước SIGKILL
$ docker start order-service                # start lại
$ docker restart order-service              # stop + start
$ docker kill order-service                 # SIGKILL ngay (brutal)
$ docker pause order-service               # tạm dừng (cgroup freeze)
$ docker unpause order-service             # tiếp tục

# ── Xóa containers ──

$ docker rm -f order-service               # force stop + remove
$ docker container prune                    # xóa tất cả stopped containers

# ── Logs ──
$ docker logs order-service                 # toàn bộ logs
$ docker logs -f order-service             # follow real-time
$ docker logs --tail 100 order-service     # 100 dòng cuối
$ docker logs --since 30m order-service    # 30 phút gần đây

# ── Copy files ──
$ docker cp order-service:/app/logs/app.log ./  # từ container ra host
$ docker cp ./config.yml order-service:/app/   # từ host vào container

Dockerfile — Tất cả Instructions quan trọng

FROM, RUN, COPY, ADD, ENV, ARG, EXPOSE, CMD, ENTRYPOINT — khi nào dùng gì

Dockerfile đầy đủ cho Java Spring Boot
Dockerfile — Spring Boot với annotations chi tiếtDockerfile
# ── Base image ──
FROM eclipse-temurin:21-jre-jammy
# Dùng JRE (không phải JDK) cho production — nhỏ hơn ~200MB
# jammy = Ubuntu 22.04 LTS (stable, security patches)

# ── Build-time arguments ──
ARG APP_VERSION=latest          # docker build --build-arg APP_VERSION=1.2.3
ARG BUILD_DATE

# ── Metadata / Labels ──
LABEL org.opencontainers.image.title="Order Service"
LABEL org.opencontainers.image.version="${APP_VERSION}"
LABEL org.opencontainers.image.created="${BUILD_DATE}"
LABEL maintainer="devops@company.com"

# ── Environment variables ──
ENV JAVA_OPTS="-XX:+UseContainerSupport -XX:MaxRAMPercentage=75.0"
ENV APP_HOME="/app"
# ENV vs ARG: ENV tồn tại khi runtime, ARG chỉ khi build

# ── Tạo non-root user (security!) ──
RUN groupadd --system --gid 1001 app \
 && useradd --system --uid 1001 --gid app --no-create-home app
# Combine RUN commands → ít layers hơn!

# ── Working directory ──
WORKDIR ${APP_HOME}
# WORKDIR tạo thư mục nếu chưa tồn tại
# Tất cả COPY, RUN, CMD sau đều relative đến đây

# ── Copy files ──
COPY --chown=app:app target/app.jar ./app.jar
# COPY vs ADD:
# COPY: đơn giản, chỉ copy local files (preferred)
# ADD: có thể untar archives, download URLs (tránh dùng)

# ── Switch to non-root user ──
USER app

# ── Expose port ──
EXPOSE 8080
# EXPOSE chỉ là documentation — không thực sự mở port
# Phải dùng -p khi docker run để map ra host

# ── Health check ──
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

# ── Entry point vs CMD ──
ENTRYPOINT ["java"]
CMD ["-jar", "app.jar"]
# ENTRYPOINT: không thể override (trừ --entrypoint)
# CMD: default args cho ENTRYPOINT, có thể override
# docker run my-app -jar app.jar --spring.profiles.active=dev

# Kết hợp cả JAVA_OPTS:
# ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar $@", "--"]
# CMD []

MULTI-STAGE

Multi-stage Build — Image nhỏ, Build nhanh

Tách build environment khỏi runtime — giảm image size từ 1GB xuống 200MB

Multi-stage cho Spring Boot — 3 stages
Dockerfile.multi-stage — dependency cache + extract layersMulti-stage
# ══════ Stage 1: Dependency cache ══════
FROM eclipse-temurin:21-jdk-jammy AS deps
WORKDIR /build

# Copy chỉ pom.xml trước → download deps → cache layer này
COPY .mvn/ .mvn/
COPY mvnw pom.xml ./
RUN ./mvnw dependency:go-offline --no-transfer-progress
# Layer này chỉ rebuild khi pom.xml thay đổi!

# ══════ Stage 2: Build ══════
FROM deps AS builder
COPY src/ src/
RUN ./mvnw package -DskipTests --no-transfer-progress

# Spring Boot layer extraction — tối ưu Docker cache
RUN java -Djarmode=layertools -jar target/*.jar extract

# ══════ Stage 3: Runtime ══════
FROM eclipse-temurin:21-jre-jammy AS runtime
# Chỉ JRE — không có Maven, JDK tools → image nhỏ hơn ~400MB

RUN groupadd -r app && useradd -r -g app app
WORKDIR /app

# Copy Spring Boot layers theo thứ tự (ít thay đổi nhất → nhiều nhất)
COPY --from=builder --chown=app:app /build/dependencies/ ./
COPY --from=builder --chown=app:app /build/spring-boot-loader/ ./
COPY --from=builder --chown=app:app /build/snapshot-dependencies/ ./
COPY --from=builder --chown=app:app /build/application/ ./
# Layer application thay đổi mỗi build → ở cuối

USER app
EXPOSE 8080
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

ENV JAVA_OPTS="-XX:+UseContainerSupport -XX:MaxRAMPercentage=75.0 -Djava.security.egd=file:/dev/./urandom"
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS org.springframework.boot.loader.launch.JarLauncher"]

So sánh Bad vs Good Dockerfile patterns
Bad vs Good practicesBest Practices
# ❌ BAD — apt không clean up, nhiều RUN layers
RUN apt update
RUN apt install -y curl
RUN apt install -y jq
# → 3 layers riêng biệt, cache apt còn lại

# ✅ GOOD — một layer, sạch cache
RUN apt-get update \
 && apt-get install -y --no-install-recommends curl jq \
 && apt-get clean \
 && rm -rf /var/lib/apt/lists/*
# → 1 layer duy nhất, không có apt cache

# ❌ BAD — copy tất cả source trước (cache miss mỗi lần)
COPY . .
RUN ./mvnw package

# ✅ GOOD — copy dependencies trước, source code sau
COPY pom.xml mvnw ./
COPY .mvn/ .mvn/
RUN ./mvnw dependency:go-offline   # cache khi pom.xml không đổi
COPY src/ src/
RUN ./mvnw package -DskipTests

# ❌ BAD — chạy root
USER root
CMD ["java", "-jar", "app.jar"]

# ✅ GOOD — non-root user
RUN useradd -r -u 1001 appuser
USER 1001
CMD ["java", "-jar", "app.jar"]

# ❌ BAD — latest tag không ổn định
FROM openjdk:latest

# ✅ GOOD — pin version cụ thể
FROM eclipse-temurin:21.0.4_7-jre-jammy
# hoặc dùng digest: FROM eclipse-temurin@sha256:abc123...

# ── Tạo custom network ──
$ docker network create backend-net
$ docker network create --driver bridge --subnet 172.20.0.0/16 backend-net

# ── Xem networks ──
$ docker network ls
$ docker network inspect backend-net

# ── Kết nối containers vào network ──
$ docker run -d --name postgres --network backend-net postgres:16
$ docker run -d --name order-service --network backend-net my-app

# ✅ Containers cùng network có thể gọi nhau bằng TÊN container!
# order-service → jdbc:postgresql://postgres:5432/shopdb ✅
# KHÔNG cần IP thật — Docker DNS tự resolve tên container sang IP

# ── Connect/disconnect container đang chạy ──
$ docker network connect backend-net existing-container
$ docker network disconnect backend-net existing-container

# ── Host network (Linux only) ──
$ docker run --network host nginx
# Container dùng port 80 của host trực tiếp, không cần -p 80:80
# ⚠️ Không hoạt động trên macOS/Windows (Docker Desktop dùng VM)

# ── Port mapping ──
$ docker run -p 8080:8080 my-app      # host:container
$ docker run -p 127.0.0.1:8080:8080 my-app  # chỉ localhost (an toàn hơn)
$ docker run -P my-app              # auto-map tất cả EXPOSE ports

Volumes & Bind Mounts — full examplesStorage
# ── Named Volumes ──
$ docker volume create pgdata
$ docker volume ls
$ docker volume inspect pgdata

# Mount named volume khi run container
$ docker run -v pgdata:/var/lib/postgresql/data postgres:16
# pgdata = tên volume, /var/lib/postgresql/data = path trong container

# ── Bind Mount ──
# Mount thư mục source code cho development
$ docker run -v /home/dev/my-app/src:/app/src my-app
# Syntax mới hơn --mount (rõ ràng hơn):
$ docker run \
    --mount type=bind,source=$(pwd)/src,target=/app/src \
    --mount type=bind,source=$(pwd)/config.yml,target=/app/config.yml,readonly \
    my-app

# ── Read-only mount (security) ──
$ docker run -v /etc/ssl/certs:/etc/ssl/certs:ro my-app

# ── Backup & Restore volumes ──
# Backup:
$ docker run --rm \
    -v pgdata:/data \
    -v $(pwd):/backup \
    alpine tar czf /backup/pgdata-backup.tar.gz -C /data .

# Restore:
$ docker run --rm \
    -v pgdata:/data \
    -v $(pwd):/backup \
    alpine tar xzf /backup/pgdata-backup.tar.gz -C /data

# ── Xóa volumes ──
$ docker volume rm pgdata              # xóa cụ thể
$ docker volume prune                  # xóa volumes không dùng

# compose.yml (Docker Compose v3.8+)

services:

  postgres:
    image:           postgres:16-alpine
    container_name: postgres
    restart:         unless-stopped
    environment:
      POSTGRES_DB:       shopdb
      POSTGRES_USER:     admin
      POSTGRES_PASSWORD: ${DB_PASSWORD}    # từ .env file
    ports:
      - "127.0.0.1:5432:5432"               # chỉ localhost
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    healthcheck:
      test:     ["CMD-SHELL", "pg_isready -U admin -d shopdb"]
      interval: 10s
      timeout:  5s
      retries:  5
    networks:
      - backend
    deploy:
      resources:
        limits:
          memory: 512m
          cpus:   "1.0"

  redis:
    image:   redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD} --appendonly yes
    volumes:
      - redisdata:/data
    networks:
      - backend
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s

  order-service:
    build:
      context:    .
      dockerfile: Dockerfile
      target:     runtime              # multi-stage target
      args:
        APP_VERSION: ${APP_VERSION:-dev}
    image:           registry.company.com/order-service:${APP_VERSION:-dev}
    container_name: order-service
    restart:         on-failure:3
    depends_on:
      postgres:
        condition: service_healthy  # chờ postgres healthy!
      redis:
        condition: service_healthy
    environment:
      SPRING_PROFILES_ACTIVE:    prod
      SPRING_DATASOURCE_URL:     jdbc:postgresql://postgres:5432/shopdb
      SPRING_DATASOURCE_USERNAME:admin
      SPRING_DATASOURCE_PASSWORD:${DB_PASSWORD}
      SPRING_REDIS_HOST:         redis
      SPRING_REDIS_PASSWORD:     ${REDIS_PASSWORD}
    ports:
      - "8080:8080"
    volumes:
      - ./logs:/app/logs
    healthcheck:
      test:          ["CMD-SHELL", "curl -f http://localhost:8080/actuator/health || exit 1"]
      interval:      30s
      timeout:       10s
      start_period:  60s
      retries:       3
    networks:
      - backend
      - frontend
    logging:
      driver:  json-file
      options:
        max-size: "100m"
        max-file: "5"

volumes:
  pgdata:    # named volume, Docker quản lý
  redisdata:

networks:
  backend:
    driver:   bridge
    internal: true   # không có external access

Security best practices
Docker security — non-root, capabilities, secretsSecurity
# ── compose.yml security settings ──
services:
  order-service:
    user: "1001:1001"              # chạy non-root
    read_only: true               # filesystem read-only
    tmpfs:
      - /tmp                       # tmp vẫn writable
      - /app/tmp
    security_opt:
      - no-new-privileges:true     # không cho escalate privileges
    cap_drop:
      - ALL                        # bỏ tất cả Linux capabilities
    cap_add:
      - NET_BIND_SERVICE           # chỉ thêm cái cần (bind port <1024)
    secrets:
      - db_password

secrets:
  db_password:
    file: ./secrets/db_password.txt  # file chứa password
    # → mount tại /run/secrets/db_password trong container
    # Đọc: db.password=@/run/secrets/db_password (Spring Boot)

$ docker image prune -a --filter "until=24h"   # images cũ hơn 24h
$ docker container prune --filter "status=exited"

# /etc/cron.weekly/docker-cleanup
#!/bin/bash
docker system prune -f
docker image prune -a -f --filter "until=168h"  # >7 ngày
docker builder prune -f --keep-storage 5g        # giữ 5GB build cache

Docker Best Practices — Checklist Production
1

Dùng official base images và pin version

Tránh :latest trên production. Dùng eclipse-temurin:21.0.4-jre-jammy thay vì java:latest. Subscribe security advisories của base image.

2

Multi-stage build — tách builder khỏi runtime

Stage builder dùng JDK, Maven, npm. Stage runtime chỉ có JRE/Node. Image cuối không có build tools, source code hay test dependencies.

3

Non-root user và minimal permissions

Tạo user system --uid 1001 trong Dockerfile. USER 1001 trước CMD/ENTRYPOINT. cap_drop: [ALL] và chỉ thêm capabilities thực sự cần.

4

Optimize layer caching — copy dependencies trước

Copy pom.xml → mvnw dependency:go-offline → copy src/ → mvnw package. Khi chỉ source thay đổi, dependency layer được cache lại.

5

HEALTHCHECK trong Dockerfile và compose.yml

Dùng HEALTHCHECK để Docker biết app thực sự ready. Compose dùng depends_on condition: service_healthy để đảm bảo thứ tự khởi động đúng.

6

Log rotation và resource limits

Cấu hình max-size: 100m, max-file: 5 cho logging. Set memory: 512m, cpus: 1.0 trong compose. Không để container dùng vô hạn tài nguyên của host.
