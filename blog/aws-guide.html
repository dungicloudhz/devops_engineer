Tại sao dùng Cloud thay vì tự đặt server?
❌ On-premise Server
💸 Mua server vài trăm triệu VND
🏢 Thuê data center, trả tiền điện
📉 Traffic thấp → lãng phí tài nguyên
📈 Traffic cao → không đủ server
🔧 Tự bảo trì phần cứng khi hỏng
⏳ Cần 2-4 tuần để thêm server mới
✅ AWS Cloud
💳 Pay-as-you-go, chỉ trả tiền dùng
🌍 Data center toàn cầu sẵn có
📉 Scale down = tiết kiệm chi phí
📈 Scale up trong vài phút tự động
🔧 AWS lo phần cứng hoàn toàn
⚡ Launch server trong 2 phút

ECS vs EKS — chọn cái nào?
🐋 Amazon ECS
AWS-native, đơn giản hơn

Services = long-running tasks

Dùng kubectl, Helm, Kustomize

✅ Ecosystem lớn, nhiều tools

❌ Phức tạp hơn ECS
Dùng khi: team lớn, multi-cloud

LAMBDA

Lambda — Serverless Functions

Chạy code không cần quản lý server — trigger by events, pay per invocation

Lambda fundamentals — khi nào dùng
🔔
Event Driven
S3 upload → resize image. API Gateway → backend. SQS → process messages.
💰
Pay per Use
Trả tiền theo số invocations và thời gian chạy (ms). Idle = $0. 1M requests/tháng free!
⚡
Auto Scale
Từ 0 đến 1000 concurrent executions tự động. Không cần cấu hình gì.
⏱️
Stateless
Mỗi invocation độc lập. Max 15 phút. Không dùng cho long-running processes.
Lambda function Java + triggers phổ biếnLambda
# ── Lambda function Java (Spring Cloud Function) ──
// Handler class
public class OrderEventHandler implements RequestHandler<SQSEvent, Void> {
    @Override
    public Void handleRequest(SQSEvent event, Context context) {
        for (SQSEvent.SQSMessage msg : event.getRecords()) {
            String body = msg.getBody();
            context.getLogger().log("Processing: " + body);
            // process order...
        }
        return null;
    }
}

# ── Deploy Lambda ──
$ aws lambda create-function \
    --function-name  order-processor \
    --runtime        java21 \
    --handler        com.company.OrderEventHandler::handleRequest \
    --zip-file       fileb://target/function.jar \
    --role           arn:aws:iam::123456:role/LambdaExecRole \
    --memory-size    512 \
    --timeout        30 \
    --environment    Variables='{SPRING_PROFILES_ACTIVE=prod}'

# ── Update code ──
$ aws lambda update-function-code \
    --function-name order-processor \
    --image-uri     123456.dkr.ecr.ap-southeast-1.amazonaws.com/order-processor:v2

🗄️ Amazon RDS

SQS & SNS

SQS & SNS — Message Queue & Pub/Sub

SQS: Queue cho async processing · SNS: Fan-out đến nhiều subscribers

SQS vs SNS — khi nào dùng cái nào?
📬 Amazon SQS
Queue — message chờ được xử lý
1 message → 1 consumer xử lý
Standard: at-least-once, unordered
FIFO: exactly-once, ordered
Max 14 ngày retention
Dùng: background jobs, decoupling
📢 Amazon SNS
Pub/Sub — 1 message → nhiều subscribers
Push-based, fan-out
Subscribers: SQS, Lambda, Email, HTTP
Topic-based filtering
Dùng: notifications, fan-out events

Lộ trình học AWS — từ zero đến production
1

Tạo tài khoản và setup IAM an toàn

Bật MFA cho root account. Tạo IAM User với AdministratorAccess. Cài AWS CLI, chạy aws configure. Thử aws sts get-caller-identity.

2

Launch EC2 và tạo VPC cơ bản

Tạo VPC với public/private subnets. Launch EC2 t3.micro Ubuntu. SSH vào. Cài nginx. Hiểu Security Groups. Tạo Elastic IP.

3

S3, RDS và kết nối ứng dụng

Tạo S3 bucket, upload files, set permissions. Tạo RDS PostgreSQL trong private subnet. Kết nối từ EC2 qua Security Group rules. Spring Boot connect được tới DB.

4

Container: ECR + ECS Fargate

Push Docker image lên ECR. Deploy lên ECS Fargate (không cần quản lý EC2). Đặt ALB phía trước. Cấu hình Task Definition với Secrets Manager cho DB password.

5

Monitoring với CloudWatch + Alerting

Bật CloudWatch Logs cho ECS. Tạo Alarms cho CPU, Error Rate. Kết nối SNS → Slack webhook. Dashboard theo dõi hệ thống. Thực hành CloudWatch Logs Insights.

6

Infrastructure as Code với Terraform

Dùng Terraform tạo lại toàn bộ infrastructure. Module hóa VPC, EKS, RDS. Multi-environment (dev/staging/prod). GitLab CI/CD tự động apply Terraform khi merge.
