Resource Group & ARM Templates

Tổ chức resources, ARM Templates / Bicep — Infrastructure as Code của Azure

Bicep — Azure native IaC (đơn giản hơn ARM JSON)
main.bicep — deploy Storage Account + App ServiceBicep
// ── main.bicep — Azure Bicep IaC ──
param location string = resourceGroup().location
param environment string = 'prod'
param appName string = 'order-service'

var storageAccountName = '${replace(appName, '-', '')}${environment}sa'
var appServicePlanName = '${appName}-${environment}-plan'

// Storage Account
resource storageAccount 'Microsoft.Storage/storageAccounts@2023-01-01' = {
  name:     storageAccountName
  location: location
  sku:      { name: 'Standard_LRS' }
  kind:     'StorageV2'
  properties: {
    supportsHttpsTrafficOnly:  true
    minimumTlsVersion:         'TLS1_2'
    allowBlobPublicAccess:     false
  }
}

// App Service Plan
resource appServicePlan 'Microsoft.Web/serverfarms@2022-09-01' = {
  name:     appServicePlanName
  location: location
  sku:      { name: 'P1v3', tier: 'PremiumV3', capacity: 2 }
  kind:     'linux'
  properties: { reserved: true }
}

output storageAccountId string = storageAccount.id
output appServicePlanId string = appServicePlan.id

// ── Deploy ──
$ az deployment group create \
    --resource-group rg-order-service-prod \
    --template-file  main.bicep \
    --parameters     environment=prod appName=order-service
$ az bicep build --file main.bicep  # compile → ARM JSON
$ az deployment group what-if --resource-group rg-prod --template-file main.bicep  # preview

Azure Functions — Serverless

Triggers & Bindings — cách Azure Functions hoạt động
Azure Functions — Java examples với TriggersFunctions
// ── HTTP Trigger — REST API serverless ──
@FunctionName("GetOrder")
public HttpResponseMessage run(
    @HttpTrigger(name = "req", methods = {HttpMethod.GET},
                 route = "orders/{orderId}",
                 authLevel = AuthorizationLevel.FUNCTION)
    HttpRequestMessage<Optional<String>> request,
    @TableInput(name = "orders", tableName = "orders",
               rowKey = "{orderId}", partitionKey = "order",
               connection = "AzureWebJobsStorage")
    Order order,
    ExecutionContext context) {
    if (order == null) {
        return request.createResponseBuilder(HttpStatus.NOT_FOUND).build();
    }
    return request.createResponseBuilder(HttpStatus.OK)
        .body(order).header("Content-Type", "application/json").build();
}

// ── Service Bus Trigger — process messages ──
@FunctionName("ProcessOrderMessage")
public void processOrder(
    @ServiceBusTrigger(name = "msg",
                       queueName = "order-processing",
                       connection = "ServiceBusConnection")
    String messageBody,
    ExecutionContext context) {
    context.getLogger().info("Processing: " + messageBody);
    orderService.process(messageBody);
}

// ── Timer Trigger — cron job ──
@FunctionName("DailyReport")
public void run(
    @TimerTrigger(name = "timer", schedule = "0 0 2 * * *")  // 2AM mỗi ngày
    String timerInfo, ExecutionContext context) {
    reportService.generateDaily();
}

Service Bus vs Event Hub — chọn đúng
📨 Azure Service Bus
Enterprise messaging
Queues (point-to-point) & Topics (pub/sub)
Sessions, Dead-letter queue, Transactions
At-least-once delivery
Max 256KB message (Premium: 100MB)
Dùng: orders, payments, workflow
🌊 Azure Event Hub
Big data streaming — Kafka-compatible
Partitions, Consumer Groups
Millions events/second
Replay events (retention 7 ngày)
Capture to Azure Storage/Data Lake
Dùng: telemetry, logs, clickstream

// Producer
@Autowired
ServiceBusSenderClient senderClient;  // Azure Spring Boot Starter

public void publishOrderCreated(Order order) {
    ServiceBusMessage message = new ServiceBusMessage(
        new ObjectMapper().writeValueAsBytes(order));
    message.setSubject("order.created");
    message.setMessageId(order.getId());
    senderClient.sendMessage(message);
}

// Consumer (listener)
@ServiceBusListener(destination = "order-processing")
public void processOrder(String messageBody) {
    Order order = objectMapper.readValue(messageBody, Order.class);
    orderService.process(order);
}

Lộ trình học Azure — từ zero đến production
1

Tạo tài khoản và explore Azure Portal

Free account: $200 credit 30 ngày + nhiều services free 1 năm. Cài Azure CLI. Tạo Resource Group đầu tiên. Thử tạo VM nhỏ và xóa đi. Hiểu billing model.

2

Azure AD, RBAC và Managed Identity

Hiểu Subscription → Resource Group hierarchy. Tạo Service Principal. Enable Managed Identity cho VM. Gán RBAC role. Đây là nền tảng security của Azure.

3

App Service và Azure Database

Deploy Spring Boot lên App Service từ Docker container. Tạo Azure Database for PostgreSQL trong VNet. Dùng Key Vault lưu DB password. Cấu hình App Settings lấy từ Key Vault.

4

VNet, NSG và Private Endpoints

Tạo VNet với public/private subnets. NSG rules. Đặt PostgreSQL sau Private Endpoint — chỉ accessible từ VNet. App Service tích hợp VNet. Hiểu Hub & Spoke pattern.

5

AKS và Azure Container Registry

Tạo ACR. Build & push Docker image. Tạo AKS cluster. Attach ACR với AKS. Deploy Helm chart. Cấu hình Workload Identity cho pods lấy secrets từ Key Vault.

6

Azure DevOps / GitHub Actions và Monitor

Tạo CI/CD pipeline tự động: test → build Docker → push ACR → deploy AKS. Cấu hình Application Insights cho distributed tracing. Tạo Alerts gửi Teams/Slack khi có lỗi.
