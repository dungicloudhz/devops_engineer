01 — Overview

Jenkins CI/CD
+ Kubernetes
Java Backend

Hướng dẫn end-to-end xây dựng pipeline CI/CD với Jenkins, từ code commit đến production deployment trên Kubernetes (AWS EKS), dành riêng cho Java/Spring Boot microservices.

🎯 Tech Stack
Java 17 + Spring Boot 3 · Maven · Jenkins (Declarative Pipeline) · Docker · AWS ECR · Kubernetes · Helm · AWS EKS · SonarQube

🏗
Jenkins Master/Agent
Master điều phối jobs, Agent (node) thực thi. Nên chạy Agent dưới dạng K8s Pod để scale động theo load — không dùng static agents.
📝
Jenkinsfile as Code
Pipeline definition lưu trong repo (Jenkinsfile). Version-controlled, review được, không phụ thuộc Jenkins UI config. Dùng Declarative syntax.
🔌
Plugin Ecosystem
Jenkins có hơn 1800 plugins. Cần chọn lọc: Docker, Kubernetes, AWS ECR, SonarQube, Credentials Binding, Blue Ocean cho UI đẹp.
📚
Shared Library
Tái sử dụng Groovy code giữa nhiều Jenkinsfile. Đặt common stages (build, push, deploy) vào library riêng, không copy-paste giữa repos.
02 — Concepts

🧠 Core Concepts

Nắm rõ các thành phần trước khi bắt đầu setup.

📝
Declarative Pipeline
JENKINSFILE SYNTAX
Cú pháp pipeline { } có cấu trúc rõ ràng, dễ đọc. Preferred hơn Scripted pipeline cho hầu hết use cases.
🤖
Jenkins Agent
JOB EXECUTOR
Node thực thi pipeline steps. Với Kubernetes plugin, Jenkins tự spin up Pod làm agent và xóa sau khi job xong.
🔐
Credentials Store
SECRET MANAGEMENT
Jenkins quản lý secrets tập trung (username/password, secret text, SSH key, kubeconfig). Inject vào pipeline qua withCredentials().
🌿
Multibranch Pipeline
BRANCH DISCOVERY
Tự động tạo pipeline cho mỗi branch/PR trong repo. Jenkinsfile phải tồn tại trong branch đó mới trigger được.
📚
Shared Library
CODE REUSE
Global library Groovy cho phép nhiều Jenkinsfile gọi chung functions. Lưu trong repo riêng, config ở Jenkins System Settings.
🧩
Stash / Unstash
ARTIFACT PASSING
Truyền files giữa các stages/nodes trong cùng pipeline. Dùng stash để lưu JAR, unstash ở stage Docker build.
⚠️ Declarative vs Scripted
Luôn dùng Declarative Pipeline (pipeline { }) cho project mới. Scripted (node { }) chỉ dùng khi cần logic phức tạp mà Declarative không support được.
03 — Installation

⚙️ Cài đặt Jenkins trên Kubernetes

Cách tốt nhất là deploy Jenkins lên chính K8s cluster bằng Helm, để Jenkins Agent cũng là K8s Pod.

1
Add Helm repo và install Jenkins
Dùng official Jenkins Helm chart để deploy lên EKS.
Shell · install Jenkins via Helm
# Thêm Jenkins Helm repo
helm repo add jenkins https://charts.jenkins.io
helm repo update

# Tạo namespace riêng
kubectl create namespace jenkins

# Install với custom values
helm install jenkins jenkins/jenkins \
  --namespace jenkins \
  --values jenkins-values.yaml \
  --wait
2
jenkins-values.yaml — cấu hình quan trọng
Config persistent storage, resource limits, và Kubernetes Cloud để spin up dynamic agents.
YAML · jenkins-values.yaml

    - git:5.2.0

3
Lấy initial admin password
Sau khi Jenkins pod chạy, lấy password để đăng nhập lần đầu.
Shell · get admin password
# Lấy password từ secret
kubectl exec --namespace jenkins -it svc/jenkins \
  -c jenkins -- /bin/cat /run/secrets/additional/chart-admin-password

# Hoặc qua Helm
printf $(kubectl get secret --namespace jenkins jenkins \
  -o jsonpath="{.data.jenkins-admin-password}" | base64 --decode);echo
04 — Plugins

🔌 Plugins Cần Thiết

Danh sách plugins must-have cho pipeline Java + Docker + K8s.

☸️
Kubernetes Plugin
Cho phép Jenkins dùng K8s Pod làm dynamic agents. Tự spin up khi có job, tự xóa sau khi xong.
🐳
Docker Pipeline
Cung cấp docker.build(), docker.withRegistry() DSL trong Jenkinsfile.
☁️
Amazon ECR Plugin
Authenticate với AWS ECR để push/pull Docker images. Tích hợp với IAM credentials.
🔍
SonarQube Scanner
Chạy code analysis và enforce quality gate. Fail build nếu code quality dưới threshold.
🔑
Credentials Binding
Inject secrets từ Jenkins Credential Store vào environment variables trong pipeline steps.
🌿
Multibranch Pipeline
Tự động discover branches/PRs và tạo pipeline tương ứng. Cần cho branch strategy.
📊
Blue Ocean
Modern UI cho Jenkins pipeline. Visualize stages, parallel steps, và console output đẹp hơn.
📧
Email Extension
Gửi notification email khi build fail/success. Config template HTML, attach test reports.
✅ Plugin as Code
Với Helm chart, list plugins trong controller.installPlugins để auto-install khi khởi động. Không cần vào UI cài tay, dễ reproduce môi trường.

06 — Jenkinsfile

📝 Jenkinsfile Declarative — Đầy đủ

Jenkinsfile hoàn chỉnh cho Java Spring Boot service với tất cả stages, error handling, và notification.

Groovy · Jenkinsfile (Declarative)
pipeline {
  // Dùng Kubernetes Pod làm agent — spin up dynamic
  agent {
    kubernetes {
      yaml """
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: maven
    image: maven:3.9-eclipse-temurin-17-alpine
    command: [sleep, infinity]
    resources:
      requests: {memory: 1Gi, cpu: 500m}
      limits:   {memory: 2Gi, cpu: "1"}
    volumeMounts:
    - name: maven-cache
      mountPath: /root/.m2
  - name: docker
    image: docker:24-dind
    securityContext:
      privileged: true
    volumeMounts:
    - name: docker-sock
      mountPath: /var/run/docker.sock
  volumes:
  - name: maven-cache
    emptyDir: {}
  - name: docker-sock
    hostPath:
      path: /var/run/docker.sock
"""
    }
  }

  environment {
    // Credentials từ Jenkins Credential Store
    AWS_CREDENTIALS    = credentials('aws-credentials')
    KUBECONFIG_DEV     = credentials('kubeconfig-dev')
    KUBECONFIG_PROD    = credentials('kubeconfig-prod')
    SONAR_TOKEN        = credentials('sonarqube-token')
    SLACK_WEBHOOK      = credentials('slack-webhook-url')

    // Computed values
    AWS_ACCOUNT_ID     = '123456789012'
    AWS_REGION         = 'ap-southeast-1'
    ECR_REGISTRY       = "${AWS_ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com"
    IMAGE_NAME         = "${ECR_REGISTRY}/${JOB_BASE_NAME}"
    IMAGE_TAG          = "${GIT_COMMIT[0..7]}"  // 8-char commit SHA
    HELM_RELEASE       = "${JOB_BASE_NAME}"
  }

  options {
    buildDiscarder(logRotator(numToKeepStr: '20'))
    timestamps()
    timeout(time: 30, unit: 'MINUTES')
    disableConcurrentBuilds()          // không chạy song song cùng branch
    ansiColor('xterm')               // colored output
  }

  stages {

    // ── Stage 1: Checkout ────────────────────────
    stage('Checkout') {
      steps {
        checkout scm
        script {
          env.GIT_COMMIT_MSG = sh(
            script: 'git log -1 --pretty=%B',
            returnStdout: true
          ).trim()
          env.GIT_AUTHOR = sh(
            script: 'git log -1 --pretty=%an',
            returnStdout: true
          ).trim()
        }
        echo "Branch: ${BRANCH_NAME} | Commit: ${IMAGE_TAG} | Author: ${GIT_AUTHOR}"
      }
    }

    // ── Stage 2: Unit Test ───────────────────────
    stage('Unit Test') {
      steps {
        container('maven') {
          sh 'mvn test -B --no-transfer-progress'
        }
      }
      post {
        always {
          junit 'target/surefire-reports/*.xml'
          jacoco(
            execPattern: 'target/*.exec',
            classPattern: 'target/classes',
            sourcePattern: 'src/main/java'
          )
        }
      }
    }

    // ── Stage 3: SonarQube Analysis ──────────────
    stage('Code Analysis') {
      steps {
        container('maven') {
          withSonarQubeEnv('SonarQube') {
            sh """
              mvn sonar:sonar \
                -Dsonar.projectKey=${JOB_BASE_NAME} \
                -Dsonar.token=${SONAR_TOKEN} \
                -B --no-transfer-progress
            """
          }
        }
      }
    }

    // ── Stage 4: Quality Gate ────────────────────
    stage('Quality Gate') {
      steps {
        timeout(time: 5, unit: 'MINUTES') {
          waitForQualityGate abortPipeline: true
        }
      }
    }

    // ── Stage 5: Build JAR ───────────────────────
    stage('Build JAR') {
      when {
        anyOf {
          branch 'main'
          branch 'develop'
          branch pattern: 'release/.*', comparator: 'REGEXP'
        }
      }
      steps {
        container('maven') {
          sh 'mvn package -DskipTests -B --no-transfer-progress'
        }
        stash includes: 'target/*.jar', name: 'app-jar'
      }
    }

    // ── Stage 6: Docker Build & Push ─────────────
    stage('Docker Build & Push') {
      when {
        anyOf {
          branch 'main'; branch 'develop'
          branch pattern: 'release/.*', comparator: 'REGEXP'
        }
      }
      steps {
        container('docker') {
          unstash 'app-jar'
          script {
            // Login ECR
            withCredentials([[
              $class: 'AmazonWebServicesCredentialsBinding',
              credentialsId: 'aws-credentials',
              accessKeyVariable: 'AWS_ACCESS_KEY_ID',
              secretKeyVariable: 'AWS_SECRET_ACCESS_KEY'
            ]]) {
              sh """
                aws ecr get-login-password --region ${AWS_REGION} | \
                  docker login --username AWS --password-stdin ${ECR_REGISTRY}
              """
            }
            // Build image
            def img = docker.build(
              "${IMAGE_NAME}:${IMAGE_TAG}",
              "--build-arg BUILD_DATE=\$(date -u +'%Y-%m-%dT%H:%M:%SZ') ."
            )
            // Push commit tag + branch slug
            img.push()
            img.push(env.BRANCH_NAME.replaceAll('/', '-'))
          }
        }
      }
    }

    // ── Stage 7: Deploy Dev ──────────────────────
    stage('Deploy Dev') {
      when { branch 'develop' }
      environment {
        KUBECONFIG = "${KUBECONFIG_DEV}"
        NAMESPACE  = 'dev'
      }
      steps {
        container('maven') {   // reuse maven container cho helm/kubectl
          sh """
            curl -fsSL https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
            helm upgrade --install ${HELM_RELEASE} ./helm \
              --namespace ${NAMESPACE} --create-namespace \
              --values ./helm/values.yaml \
              --values ./helm/values-dev.yaml \
              --set image.repository=${IMAGE_NAME} \
              --set image.tag=${IMAGE_TAG} \
              --atomic --timeout 5m --wait
          """
        }
      }
      post {
        success { sh './jenkins/smoke-test.sh dev' }
      }
    }

    // ── Stage 8: Deploy Prod (manual gate) ───────
    stage('Approve Production') {
      when { branch 'main' }
      steps {
        timeout(time: 1, unit: 'HOURS') {
          input message: "Deploy ${IMAGE_TAG} to Production?",
                ok: 'Deploy',
                submitter: 'admin,lead-devs'
        }
      }
    }

    stage('Deploy Production') {
      when { branch 'main' }
      environment {
        KUBECONFIG = "${KUBECONFIG_PROD}"
        NAMESPACE  = 'production'
      }
      steps {
        container('maven') {
          sh """
            helm upgrade --install ${HELM_RELEASE} ./helm \
              --namespace ${NAMESPACE} \
              --values ./helm/values.yaml \
              --values ./helm/values-prod.yaml \
              --set image.repository=${IMAGE_NAME} \
              --set image.tag=${IMAGE_TAG} \
              --atomic --timeout 10m --wait
          """
        }
      }
    }

  } // end stages

  post {
    success {
      script { notifySlack('SUCCESS', env.SLACK_WEBHOOK) }
    }
    failure {
      script { notifySlack('FAILURE', env.SLACK_WEBHOOK) }
    }
    always {
      cleanWs()   // dọn workspace sau mỗi build
    }
  }

} // end pipeline

🧪 Parallel Test + Analysis

Chạy song song unit test và integration test để giảm thời gian pipeline.

Groovy · parallel stages
stage('Test') {
  parallel {
    stage('Unit Test') {
      steps {
        container('maven') {
          sh 'mvn test -Dtest="**/*Test" -B --no-transfer-progress'
        }
      }
      post {
        always {
          junit 'target/surefire-reports/*.xml'
        }
      }
    }
    stage('Integration Test') {
      steps {
        container('maven') {
          sh """
            mvn verify -Dtest="**/*IT" \
              -Dspring.datasource.url=jdbc:postgresql://postgres:5432/testdb \
              -B --no-transfer-progress
          """
        }
      }
      post {
        always {
          junit 'target/failsafe-reports/*.xml'
        }
      }
    }
  }
}
💡 Sidecar containers trong Kubernetes Pod
Muốn chạy PostgreSQL cho integration test, thêm container postgres:15-alpine vào YAML spec của Kubernetes agent. Các containers trong cùng Pod chia sẻ localhost network.
Groovy · agent pod với postgres sidecar
agent {
  kubernetes {
    yaml """
apiVersion: v1
kind: Pod
spec:
  containers:
  - name: maven
    image: maven:3.9-eclipse-temurin-17-alpine
    command: [sleep, infinity]
  - name: postgres          # Sidecar cho integration test
    image: postgres:15-alpine
    env:
    - name: POSTGRES_DB
      value: testdb
    - name: POSTGRES_USER
      value: test
    - name: POSTGRES_PASSWORD
      value: test
    ports:
    - containerPort: 5432
"""
  }
}

08 — Credentials

🔑 Credentials & Secrets

Jenkins quản lý secrets qua Manage Jenkins → Credentials. Không bao giờ hardcode trong Jenkinsfile.

Credential IDTypeMô tả
aws-credentialsAWS CredentialsAccess Key + Secret Key để push ECR, EKS access
kubeconfig-devSecret filekubectl config file cho dev cluster
kubeconfig-prodSecret filekubectl config file cho production cluster
sonarqube-tokenSecret textToken authenticate SonarQube server
slack-webhook-urlSecret textSlack incoming webhook URL
docker-registryUsername/PasswordRegistry credentials nếu không dùng ECR

🔐 Cách sử dụng trong pipeline

Groovy · withCredentials patterns
// Pattern 1: AWS Credentials
withCredentials([[
  $class: 'AmazonWebServicesCredentialsBinding',
  credentialsId: 'aws-credentials',
  accessKeyVariable: 'AWS_ACCESS_KEY_ID',
  secretKeyVariable: 'AWS_SECRET_ACCESS_KEY'
]]) {
  sh 'aws ecr get-login-password ...'
}

// Pattern 2: Secret text
withCredentials([string(credentialsId: 'sonarqube-token', variable: 'SONAR_TOKEN')]) {
  sh 'mvn sonar:sonar -Dsonar.token=${SONAR_TOKEN}'
}

// Pattern 3: Secret file (kubeconfig)
withCredentials([file(credentialsId: 'kubeconfig-prod', variable: 'KUBECONFIG')]) {
  sh 'kubectl get pods -n production'
}

// Pattern 4: Username + Password
withCredentials([usernamePassword(
  credentialsId: 'docker-registry',
  usernameVariable: 'DOCKER_USER',
  passwordVariable: 'DOCKER_PASS'
)]) {
  sh 'docker login -u $DOCKER_USER -p $DOCKER_PASS'
}

// Pattern 5: Khai báo trong environment block
environment {
  MY_SECRET = credentials('my-secret-id')
  // Jenkins tự tạo MY_SECRET_USR và MY_SECRET_PSW nếu là username/pass
}
⚠️ Secrets bị mask nhưng không hoàn toàn an toàn
Jenkins mask secret values trong logs nhưng nếu app in ra từng ký tự riêng lẻ thì vẫn lộ. Không echo $SECRET và không print object chứa secret. Dùng set +x trước khi chạy lệnh có secrets.
09 — Docker

🐳 Dockerfile Multi-Stage cho Spring Boot

Dockerfile · optimized multi-stage
# ── Stage 1: Dependency Cache ──────────────────
FROM maven:3.9-eclipse-temurin-17-alpine AS deps
WORKDIR /app
# Copy pom.xml trước để cache deps layer
COPY pom.xml .
RUN mvn dependency:go-offline -B -q

# ── Stage 2: Build ─────────────────────────────
FROM deps AS builder
COPY src ./src
RUN mvn package -DskipTests -B -q

# ── Stage 3: Layer Extract (Spring Boot) ───────
FROM eclipse-temurin:17-jre-alpine AS extractor
WORKDIR /app
COPY --from=builder /app/target/*.jar app.jar
RUN java -Djarmode=layertools -jar app.jar extract

# ── Stage 4: Runtime ───────────────────────────
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app

# Security: non-root user
RUN addgroup -S app && adduser -S app -G app
USER app

# Copy layers từ ít thay đổi → nhiều thay đổi
COPY --from=extractor --chown=app:app /app/dependencies/ ./
COPY --from=extractor --chown=app:app /app/spring-boot-loader/ ./
COPY --from=extractor --chown=app:app /app/snapshot-dependencies/ ./
COPY --from=extractor --chown=app:app /app/application/ ./

# Build metadata labels
ARG BUILD_DATE
ARG GIT_COMMIT
LABEL org.opencontainers.image.created="${BUILD_DATE}"
LABEL org.opencontainers.image.revision="${GIT_COMMIT}"

EXPOSE 8080
ENTRYPOINT ["java", "-XX:MaxRAMPercentage=75", "org.springframework.boot.loader.JarLauncher"]

☸️ Kubernetes Manifests

📦 Deployment với Rolling Update

YAML · deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-service
  namespace: production
  annotations:
    deployment.kubernetes.io/revision: "1"
    jenkins.build: "{{ JENKINS_BUILD_URL }}"
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-service
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: my-service
        version: "{{ IMAGE_TAG }}"
    spec:
      terminationGracePeriodSeconds: 60
      containers:
        - name: my-service
          image: "{{ IMAGE_NAME }}:{{ IMAGE_TAG }}"
          ports:
            - containerPort: 8080
          envFrom:
            - configMapRef:
                name: my-service-config
          env:
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: my-service-secret
                  key: db-password
          resources:
            requests:
              memory: "512Mi"
              cpu: "250m"
            limits:
              memory: "1Gi"
              cpu: "500m"
          startupProbe:        # tốt hơn initialDelaySeconds
            httpGet:
              path: /actuator/health
              port: 8080
            failureThreshold: 30   # 30 * 10s = 5 phút max startup
            periodSeconds: 10
          livenessProbe:
            httpGet:
              path: /actuator/health/liveness
              port: 8080
            periodSeconds: 15
            failureThreshold: 3
          readinessProbe:
            httpGet:
              path: /actuator/health/readiness
              port: 8080
            periodSeconds: 10
          lifecycle:
            preStop:             # Graceful shutdown
              exec:
                command: ["/bin/sh", "-c", "sleep 10"]
✅ startupProbe vs initialDelaySeconds
Dùng startupProbe thay vì initialDelaySeconds cứng. startupProbe cho phép app startup chậm (Liquibase migration, bean initialization) mà không ảnh hưởng livenessProbe sensitivity sau khi up.

Groovy · shared library function
/**
 * Build Docker image và push lên ECR.
 * @param config Map: [imageName, imageTag, awsRegion, awsCredId]
 */
def call(Map config) {
  def imageName  = config.imageName
  def imageTag   = config.imageTag
  def awsRegion  = config.get('awsRegion', 'ap-southeast-1')
  def credId     = config.get('awsCredId', 'aws-credentials')

  container('docker') {
    withCredentials([[
      $class: 'AmazonWebServicesCredentialsBinding',
      credentialsId: credId,
      accessKeyVariable: 'AWS_ACCESS_KEY_ID',
      secretKeyVariable: 'AWS_SECRET_ACCESS_KEY'
    ]]) {
      def ecrHost = sh(
        script: "aws sts get-caller-identity --query Account --output text",
        returnStdout: true
      ).trim() + ".dkr.ecr.${awsRegion}.amazonaws.com"

      sh """
        aws ecr get-login-password --region ${awsRegion} | \\
          docker login --username AWS --password-stdin ${ecrHost}
      """

      def img = docker.build("${imageName}:${imageTag}")
      img.push()
      img.push('latest')

      echo "✅ Pushed: ${imageName}:${imageTag}"
    }
  }
}

def getDeployConfig(String branchName) {
  if (branchName == 'main') {
    return [env: 'prod', ns: 'production', kube: 'kubeconfig-prod', auto: false]
  } else if (branchName == 'develop') {
    return [env: 'dev', ns: 'dev', kube: 'kubeconfig-dev', auto: true]
  } else if (branchName.startsWith('release/')) {
    return [env: 'staging', ns: 'staging', kube: 'kubeconfig-dev', auto: true]
  }
  return null   // feature branches → không deploy
}

stage('Deploy') {
  steps {
    script {
      def cfg = getDeployConfig(env.BRANCH_NAME)
      if (cfg == null) {
        echo "Skipping deploy for branch: ${BRANCH_NAME}"
        return
      }
      if (!cfg.auto) {
        input message: "Deploy to ${cfg.env}?", submitter: 'admin'
      }
      deployToK8s(
        release: 'my-service',
        namespace: cfg.ns,
        imageRepo: env.IMAGE_NAME,
        imageTag: env.IMAGE_TAG,
        valuesFile: "values-${cfg.env}.yaml",
        kubeconfigId: cfg.kube
      )
    }
  }
}

18 — Tips & Pitfalls

💡 Tips, Tricks & Common Pitfalls

🔴 Pitfall #1 — DinD (Docker in Docker)
Tránh dùng Docker in Docker (privileged DinD container) trong Kubernetes agent vì security risk và instability. Thay bằng mount /var/run/docker.sock từ host node, hoặc tốt hơn là dùng Kaniko để build image không cần Docker daemon.
🔴 Pitfall #2 — Static Jenkins Agents
Đừng dùng static agent nodes (on-premise VMs). Kubernetes dynamic agents spin up Pod khi cần, scale to zero khi idle, tự clean up → tiết kiệm resource và không bị "dirty state" từ build trước.
⚠️ Pitfall #3 — Image tag :latest
Không bao giờ dùng :latest trong deployment. Dùng commit SHA (GIT_COMMIT[0..7]) để mỗi deploy traceable và rollback được về đúng version.
⚠️ Pitfall #4 — Lưu state trong workspace
Jenkins workspace có thể bị reuse giữa các builds. Dùng cleanWs() trong post { always { } } để dọn sạch. Dùng stash/unstash để truyền artifacts, không dựa vào filesystem persistence.
⚠️ Pitfall #5 — Groovy Script Approval
Jenkins sandbox block một số Groovy methods. Khi gặp Scripts not permitted to use method..., cần Admin approve ở Manage Jenkins → In-process Script Approval. Hoặc dùng Shared Library (trusted code) để tránh sandbox restriction.

✅ Best Practices Tổng Hợp

  • Declarative Pipeline — dùng pipeline { } syntax, dễ đọc và validate hơn Scripted
  • Kubernetes dynamic agents — không static agents, scale to zero khi idle
  • Shared Library — common logic một chỗ, không copy-paste giữa repos
  • credentials() trong environment block — Jenkins auto-mask, không echo ra logs
  • stash/unstash — truyền artifacts (JAR) giữa containers trong cùng pipeline
  • --atomic trong Helm — auto-rollback khi deployment fail, không cần manual
  • startupProbe cho Spring Boot — flexible hơn initialDelaySeconds cứng
  • disableConcurrentBuilds() — tránh race condition khi 2 builds cùng deploy lên 1 env
  • timeout() bao quanh input — pipeline không bị treo mãi chờ approval, tự fail sau X giờ
  • cleanWs() trong post always — không để dirty files ảnh hưởng build sau
  • numToKeepStr: '20' — giới hạn build history để không tốn disk Jenkins master
